# PurpleTrace Lab v0.2.0 Release Notes

PurpleTrace Lab v0.2.0 expands the project from a basic detection pipeline into a more complete defensive detection engineering lab.

This release focuses on safe synthetic telemetry generation, richer alert outputs, local visualization, and analyst-style investigation workflow.

---

## Release Summary

PurpleTrace Lab v0.2.0 includes:

* Synthetic telemetry simulator
* Expanded detection rule coverage
* Local browser-based dashboard
* Investigation workflow documentation
* Automatic investigation markdown exporter
* Rule coverage documentation
* Multiple report output formats
* MITRE ATT&CK metadata support
* Safe portfolio-ready demo flow

---

## Main Demo Flow

The recommended v0.2.0 demo flow is:

```text
Simulator -> Agent -> Reports -> Dashboard -> Investigation Report
```

This means:

1. Generate safe synthetic endpoint telemetry.
2. Analyze telemetry using JSON-based detection rules.
3. Export alerts in multiple formats.
4. Load alert JSON into the local dashboard.
5. Review investigation-ready markdown reports.

---

## New and Improved Components

### 1. PurpleTrace Simulator

The simulator generates safe synthetic endpoint telemetry.

Example:

```powershell
dotnet run --project src\PurpleTrace.Simulator -- --scenario all --format jsonl --out samples\simulated-events.local.jsonl
```

Supported scenarios include:

* PowerShell execution
* Encoded PowerShell command
* Certutil download pattern
* Registry discovery
* Rundll32 URL handler usage
* Service discovery

The simulator does not execute commands or modify the system. It only writes static JSON or JSONL telemetry files.

---

### 2. PurpleTrace Agent

PurpleTrace Agent analyzes endpoint-style telemetry using JSON detection rules.

Example:

```powershell
dotnet run --project src\PurpleTrace.Agent -- --source sample --rules rules --event samples\simulated-events.local.jsonl --out samples\simulator-alerts.local.json --report samples\simulator-report.local.md --csv samples\simulator-alerts.local.csv --html samples\simulator-report.local.html --summary samples\simulator-summary.local.json --investigation samples\simulator-investigation.local.md
```

Agent supports:

* JSON alert export
* Markdown report export
* CSV alert export
* HTML report export
* JSON run summary export
* Markdown investigation report export
* Severity filtering
* MITRE technique filtering
* Rule ID filtering
* Rule tag filtering
* Rule validation
* Rule catalog listing
* Rule coverage export

---

### 3. Local Detection Dashboard

v0.2.0 adds a local browser dashboard.

Open it with:

```powershell
start dashboard\index.html
```

Then load an alert JSON file such as:

```text
samples\simulator-alerts.local.json
```

The dashboard displays:

* Total alerts
* High alerts
* MITRE technique count
* Detection rule count
* Severity distribution
* MITRE technique distribution
* Rule distribution
* Searchable alert cards
* Command-line evidence
* Evidence summaries
* Rule tags

The dashboard runs locally and does not upload data anywhere.

### 5. Investigation Markdown Exporter

Example:

The generated report includes structured cases for each alert:

* Case summary
* Alert details
* MITRE mapping
* Rule context
* Process evidence
* Matched evidence
* Source event review
* Analyst questions
* Recommended next steps
* Final decision section

## Detection Rule Coverage

v0.2.0 includes 8 detection rules.

Current rule coverage includes:

* Suspicious PowerShell execution
* Command shell started PowerShell
* Windows discovery commands
* Encoded PowerShell command
* Certutil download pattern
* Registry discovery command
* Rundll32 URL handler usage
* Windows service discovery command

Rule coverage can be exported with:

```powershell
dotnet run --project src\PurpleTrace.Agent -- --export-rule-coverage docs\rule-coverage.md --rules rules
```

## Safety Scope

PurpleTrace Lab v0.2.0 remains defensive-only.

It does not include:

* Malware
* Exploit code
* Shellcode
* Process injection
* Credential theft
* Persistence creation
* Evasion implementation
* Destructive behavior

The project focuses on safe detection engineering, telemetry analysis, alert reporting, and investigation workflow.
